# AI_LOG.md

## AI Collaboration Log

### AI Tech Stack

The application was built using a combination of AI-assisted development tools:

* Claude (Anthropic) – Primary implementation assistant used for backend architecture, API development, frontend implementation, debugging, and Docker configuration.
* ChatGPT (OpenAI) – Used for architecture validation, implementation reviews, debugging assistance, and cross-checking design decisions.

Rather than relying on AI to generate the entire application autonomously, I used the models as collaborative engineering assistants. Every generated file was reviewed, tested, and adjusted before being committed.

---

## The Prompts That Shipped It

### Backend Development

The initial backend prompt was:

> Build a simple uptime monitoring backend using Node.js, Express, TypeScript, MongoDB, and Mongoose.
>
> Requirements:
>
> * Create, list, and delete monitors
> * Ping each URL every minute
> * Store HTTP status code, response time, and timestamp
> * Keep the architecture simple and suitable for monitoring a few dozen URLs
> * Use TypeScript throughout
> * Generate production-ready folder structure

After the architecture was generated, I switched to file-by-file implementation to maintain control over the codebase.

Example follow-up prompt:

> Give me only the backend files needed next. Do not generate documentation. Show the exact folder structure, dependencies, and complete code for each file.

---

### Frontend Development

For the frontend, I supplied my actual Next.js project structure rather than using a generic scaffold.

Example prompt:

> I already have a Next.js frontend. Here is my current folder structure. Generate only the files that need to be added or modified so they fit into this structure correctly.

Additional prompt:

> Build a dashboard that displays:
>
> * Monitored URLs
> * Current UP/DOWN status
> * Latest response time
> * Last checked timestamp
>   Keep the UI simple and functional rather than over-designed.

---

### Docker & Local Development

Containerization was intentionally postponed until the application was fully functional.

Prompt:

> The frontend and backend are already working and deployed. Before generating Docker files, tell me exactly what project information you need so the Docker configuration matches the real application instead of making assumptions.

After supplying the actual package.json files, entry points, and environment variables, Docker files were generated and adjusted accordingly.

## Course Corrections

### 1. TypeScript File-Casing Conflict

During development, two model files existed simultaneously:

* Check.ts
* check.ts

Because Windows uses a case-insensitive filesystem, the issue was not immediately obvious. However, TypeScript detected conflicting module references during compilation.

Resolution:

* Removed the duplicate file
* Standardized naming conventions
* Revalidated imports across the project

---

### 2. AI-Assisted Docker Configuration

An early Docker configuration generated by AI assumed build paths and environment variables that did not match the actual project structure.

Instead of blindly using the generated configuration, I first completed deployment of the frontend and backend independently, then regenerated the Docker files using:

* Actual package.json files
* Real application entry points
* Confirmed environment variables

This produced a working Docker Compose setup aligned with the real codebase.

---

### 3. Abandoning an Autonomous AI Workflow

At the beginning of the assignment, I experimented with an autonomous multi-stage AI workflow intended to generate architecture, backend code, frontend code, Docker configuration, and documentation automatically.

After several attempts, the generated output was not reliable enough to ship.

Instead of spending additional time debugging the AI workflow itself, I switched to direct collaboration with AI tools using smaller, controlled prompts and iterative reviews.

This approach proved significantly more reliable and ultimately resulted in the delivered application.
